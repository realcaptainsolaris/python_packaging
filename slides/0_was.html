Was ist eine Abhängigkeit?

Definition: Alles, was außerhalb deines Codes liegt und notwendig ist, damit dieser wie erwartet funktioniert.

Projektspezifische Pakete

  • Typische Python-Bibliotheken
  • Installiert mit pip oder anderen Paketmanagern
  • Beziehen sich direkt auf das Projekt

Systempakete

  • Globale Pakete oder Bibliotheken
  • Installiert mit apt, homebrew, etc.
  • Von allen Nutzern und Projekten gemeinsam genutzt

Betriebssystem

  • Python läuft dank Plattform-Kompatibilität
  • Bibliotheken können OS-spezifisch sein
  • Systemoperationen variieren je nach OS

Hardware

  • Abhängigkeit von Architektur (z. B. x86, arm64)
  • Code kann für spezifische Hardware kompiliert sein
  • Beispiele: GPUs zur Beschleunigung

Die Umgebung

  • Externe Ressourcen wie APIs oder Datenbanken
  • Netzwerkzugriff ist oft erforderlich
  • Außerhalb des eigenen Computers liegende Faktoren

Die Komplexität erkennen

  • Dein Skript basiert auf einem komplexen Stack
  • Lokale Entwicklung maskiert diese Komplexität
  • Wird relevant bei der Nutzung auf anderen Rechnern

Zusammenfassung

Abhängigkeiten reichen von Paketen bis zur Hardware:

  • Projektspezifische Pakete
  • Systempakete
  • Betriebssystem und Hardware
  • Umgebung und externe Ressourcen
